\begin{abstract}
When two vision-language models (VLMs) are applied to GUI grounding, an oracle that always selects the correct model achieves 85.1\% on ScreenSpot-Pro, 5 points above the best single model (80.1\%). How much of this oracle headroom can be captured without training? We observe that multi-step zoom-in pipelines contain a free confidence signal: \emph{zoom consistency}, the distance between a model's step-2 prediction and the crop center. We prove this quantity is proportional to step-1 spatial error and show it correlates significantly with prediction correctness (Spearman $\rho = -0.14$, $p < 10^{-6}$). Using zoom consistency to route between a specialized model (KV-Ground-8B) and a general-purpose model (Qwen3.5-27B), we capture 16.5\% of the oracle headroom, achieving 80.9\% on ScreenSpot-Pro. We derive conditions under which consistency-based routing provably improves over the stronger model alone, and show that the current capture rate is limited by the generalist model's accuracy. Code is available at \url{https://github.com/omxyz/zoom-consistency-routing}.
\end{abstract}

\section{Introduction}

GUI grounding is the task of predicting click coordinates for a UI element described by a natural language instruction in a screenshot. The current best approach on ScreenSpot-Pro~\cite{li2025screenspotpro} uses KV-Ground-8B~\cite{kvground}, a specialized VLM with a 2-step zoom-in strategy, achieving 80.5\% accuracy.

Different VLMs fail on different samples. KV-Ground-8B excels on professional tools (CAD, IDEs), while Qwen3.5-27B~\cite{qwen35}, a general-purpose VLM, occasionally succeeds on consumer interfaces where the specialist fails. An oracle that always picks the correct model achieves 85.1\% on ScreenSpot-Pro, revealing 5 percentage points of untapped potential. The question is: how much of this oracle headroom can be captured at inference time, without training a router?

We address this question through \emph{zoom consistency}, a confidence signal embedded in multi-step zoom-in pipelines. When a model's step-2 prediction on a zoomed crop lands near the crop center, the model's initial localization was already accurate. When step-2 predicts far from center, step-1 was off. We formalize this signal, prove it is proportional to step-1 spatial error (Proposition~\ref{prop:linear}), and show it enables cross-model comparison without calibration (Proposition~\ref{prop:calibration}).

Using zoom consistency to route between KV-Ground-8B and Qwen3.5-27B, we capture 16.5\% of the oracle headroom (13 of 79 recoverable samples), achieving 80.9\%. We derive the necessary and sufficient condition for this routing to improve over the stronger model alone (Theorem~\ref{thm:routing}), and analyze why the capture rate is bounded by the generalist's standalone accuracy.

Our contributions:
\begin{enumerate}
    \item We formalize zoom consistency as a confidence signal and prove it is a linear estimator of step-1 spatial error (Proposition~\ref{prop:linear}).
    \item We derive the routing condition: the router improves iff its precision on the disagreement set exceeds $1/2$ (Theorem~\ref{thm:routing}).
    \item We validate the signal statistically (Spearman $\rho = -0.14$, $p < 10^{-6}$) and measure its oracle extraction rate (16.5\%) on 1,581 samples.
\end{enumerate}

\section{Related Work}

\paragraph{GUI Grounding.}
SeeClick~\cite{seeclick} established the first realistic GUI grounding benchmark. OS-Atlas~\cite{osatlas} and UGround~\cite{uground} provided large-scale training data. GUI-Actor~\cite{guiactor} introduced coordinate-free grounding. UI-TARS~\cite{uitars} scaled to 72B parameters. MEGA-GUI~\cite{megagui} proposed multi-stage coarse-to-fine grounding. KV-Ground-8B~\cite{kvground} achieved state-of-the-art results through multi-stage fine-tuning from GUI-Owl-1.5~\cite{guiowl,mplugowl}.

\paragraph{Zoom-In Strategies.}
ZoomClick~\cite{zoomclick} systematically evaluated zoom refinement. LASER~\cite{laser} introduced preference optimization for zoom region selection. ScreenSeekeR~\cite{screenseeker} used cascaded visual search. AdaZoom-GUI~\cite{adazoomgui} proposed adaptive zoom with instruction refinement. GUI-G$^2$~\cite{guig2} used Gaussian reward modeling for zoom-based grounding RL.

\paragraph{Confidence Estimation and Model Routing.}
GUI-RC~\cite{guirc} introduced region consistency via spatial voting across multiple samples. Adaptive VLM Routing~\cite{avr} proposed routing based on action difficulty estimation. Training-Free Uncertainty Guidance~\cite{uncertaintyguidance} uses model uncertainty for visual input selection. Adaptive Chain-of-Focus~\cite{adaptivecof} dynamically decides when to zoom. Our approach extracts confidence from the existing zoom pipeline with no additional forward passes.

\section{Method}

\subsection{Background: 2-Step Zoom-In Pipeline}

Given a screenshot $I$ of size $W \times H$ and instruction $q$, the VLM predicts $\hat{p}_1 = (x_1, y_1)$ in a $1000\times1000$ coordinate space. A crop of side $r \cdot 1000$ centered at $\hat{p}_1$ is resized back to $1000\times1000$, and the VLM predicts $\hat{p}_2 = (x_2, y_2)$ in the crop's space. The coordinate mapping from original to crop space is $\phi(\mathbf{q}) = (\mathbf{q} - \hat{p}_1)/r + \mathbf{m}$, where $\mathbf{m} = (500, 500)$.

\subsection{Zoom Consistency}

We define \emph{zoom consistency} as:
\begin{equation}
    c = \|\hat{p}_2 - \mathbf{m}\|_2 = \sqrt{(x_2 - 500)^2 + (y_2 - 500)^2}
\end{equation}

\begin{proposition}[Zoom Consistency as Linear Error Estimator]\label{prop:linear}
Let $\mathbf{t}$ be the true target and $\hat{p}_1 = \mathbf{t} + \boldsymbol{\varepsilon}_1$ be the step-1 prediction with error $\boldsymbol{\varepsilon}_1$. If the target lies within the crop and step-2 perfectly identifies the target ($\hat{p}_2 = \phi(\mathbf{t})$), then:
\begin{equation}
    c = \frac{\|\boldsymbol{\varepsilon}_1\|_2}{r}
\end{equation}
\end{proposition}

\begin{proof}
Under perfect step-2, $\hat{p}_2 = \phi(\mathbf{t}) = (\mathbf{t} - \hat{p}_1)/r + \mathbf{m} = -\boldsymbol{\varepsilon}_1/r + \mathbf{m}$. Therefore $\hat{p}_2 - \mathbf{m} = -\boldsymbol{\varepsilon}_1/r$ and $c = \|\boldsymbol{\varepsilon}_1\|/r$.
\end{proof}

For $r = 0.5$, the crop magnifies step-1 error by $2\times$: a consistency of $c = 100$ corresponds to step-1 error of 50 pixels (5\% of image). The linear relationship breaks when: (i) the target is outside the crop ($\|\boldsymbol{\varepsilon}_1\|_\infty > r \cdot 500$), (ii) step-2 has its own error $\boldsymbol{\varepsilon}_2$, adding noise $|c - \|\boldsymbol{\varepsilon}_1\|/r| \leq \|\boldsymbol{\varepsilon}_2\|$, or (iii) the crop is clipped at image boundaries.

\begin{proposition}[Cross-Model Comparability]\label{prop:calibration}
Let $M_A$ and $M_B$ be VLMs with different tokenizers $\mathcal{T}_A, \mathcal{T}_B$. Their zoom consistency values $c_A, c_B \in [0, 500\sqrt{2}]$ are directly comparable without calibration. In contrast, log-probability scores $\ell_A, \ell_B$ depend on token count ($L_A \neq L_B$ for identical coordinate strings), softmax temperature, and vocabulary size, requiring per-model calibration functions $g_i: \mathbb{R} \to [0,1]$ learned on held-out data.
\end{proposition}

This comparability is what enables the uncalibrated routing rule $\arg\min_i c_i$: both models output predictions in the same $1000\times1000$ coordinate space, so the consistency comparison is a geometric operation in shared units.

\subsection{Routing and Oracle Extraction}

Given models $M_A$ (specialist, accuracy $a_A$) and $M_B$ (generalist, accuracy $a_B < a_A$), partition the $N$ samples into $\mathcal{S}_{11}$ (both correct, $n_{11}$), $\mathcal{S}_{10}$ (only $A$, $n_{10}$), $\mathcal{S}_{01}$ (only $B$, $n_{01}$), $\mathcal{S}_{00}$ (both wrong, $n_{00}$). The oracle achieves $(n_{11} + n_{10} + n_{01})/N$. The \emph{oracle headroom} is $n_{01}/N$: samples only $B$ gets right.

\begin{theorem}[Routing Improvement Condition]\label{thm:routing}
A consistency-based router that selects $\arg\min_i c_i$ improves over $M_A$ alone if and only if:
\begin{equation}
    \underbrace{f_{01} \cdot n_{01}}_{\text{gains}} > \underbrace{(1 - f_{10}) \cdot n_{10}}_{\text{losses}}
\end{equation}
where $f_{10} = P(\text{router picks } A \mid \mathcal{S}_{10})$ and $f_{01} = P(\text{router picks } B \mid \mathcal{S}_{01})$. Equivalently, the router's precision in selecting $B$ on the disagreement set $\mathcal{D} = \mathcal{S}_{10} \cup \mathcal{S}_{01}$ must exceed $1/2$.
\end{theorem}

\begin{proof}
The router's accuracy is $a_{\text{rt}} = (n_{11} + f_{10} n_{10} + f_{01} n_{01})/N$. The condition $a_{\text{rt}} > a_A = (n_{11} + n_{10})/N$ reduces to $f_{01} n_{01} > (1 - f_{10}) n_{10}$, i.e., gains exceed losses. Setting $g = f_{01} n_{01}$, $\ell = (1-f_{10}) n_{10}$, the condition $g > \ell$ is $g/(g+\ell) > 1/2$.
\end{proof}

\begin{corollary}
The base rate of $\mathcal{S}_{01}$ in $\mathcal{D}$ is $\pi = n_{01}/(n_{10} + n_{01})$. The router requires a precision lift of at least $(1/2)/\pi$ over the base rate. In the observed data ($n_{10} = 304$, $n_{01} = 79$, $\pi = 0.206$), precision must exceed $2.4\times$ the base rate.
\end{corollary}

We define the \emph{oracle extraction rate} as the fraction of oracle headroom captured: $\eta = \text{net gain} / n_{01}$. A perfect router achieves $\eta = 1$; the consistency router achieves $\eta = 13/79 = 16.5\%$.

\subsection{Optimal Crop Ratio}

\begin{proposition}[Coverage-Magnification Trade-off]\label{prop:crop}
Let $\boldsymbol{\varepsilon}_1 \sim \mathcal{N}(\mathbf{0}, \sigma^2 \mathbf{I}_2)$. The probability the target lies within the crop is:
\begin{equation}
P_{\mathrm{in}}(r, \sigma) = \left[2\Phi\!\left(\frac{r}{2\sigma}\right) - 1\right]^2
\end{equation}
The effective magnification $P_{\mathrm{in}}/r$ is maximized at $r^* = 2.80\sigma$, with $P_{\mathrm{in}}(r^*) = 70.3\%$.
\end{proposition}

For $\sigma = 0.05$ (typical for strong grounding models), $r = 0.5$ provides $5\sigma$ coverage ($P_{\text{in}} \approx 100\%$) with $2\times$ magnification. An aggressive $r = 0.3$ gives $3.3\times$ magnification with $0.5\%$ target loss.

\section{Experimental Setup}

\paragraph{Benchmark.}
ScreenSpot-Pro~\cite{li2025screenspotpro,screenspotpro_repo}: 1,581 samples across 26 professional applications, 6 categories, 3 operating systems. Accuracy: point-in-box.

\paragraph{Models.}
KV-Ground-8B~\cite{kvground} (specialist, bf16, max\_pixels $= 99$M) and Qwen3.5-27B~\cite{qwen35} (generalist, AWQ-4bit via compressed\_tensors). Both use SDPA attention, 2-step zoom with $r = 0.5$, greedy decoding. Hardware: NVIDIA H200 141GB.

\paragraph{Baselines.}
KV-Ground-8B alone (current leaderboard \#1), Qwen3.5-27B alone, stage split (KV step-1, Qwen step-2), midpoint fusion, vote with agreement, KV-default fallback, and oracle.

\section{Analysis: Bottlenecks to Oracle Extraction}

\subsection{Why 16.5\%?}

By Theorem~\ref{thm:routing}, the router must achieve precision $> 1/2$ on the disagreement set $\mathcal{D}$ ($|\mathcal{D}| = 383$, base rate $\pi = 20.6\%$). The observed router has 48 gains and 35 losses (precision = 57.8\%, net +13). Three factors limit the extraction rate:

\textbf{Weak generalist.} Qwen3.5-27B achieves 60.9\% standalone accuracy, contributing only 79 unique wins. A generalist at 75\% accuracy would increase $n_{01}$ and widen the oracle headroom.

\textbf{Weak correlation.} The Spearman $\rho \approx -0.13$ means zoom consistency is a noisy discriminator. On the disagreement set, the signal must separate minority-class samples ($\mathcal{S}_{01}$, 20.6\%) from majority-class samples ($\mathcal{S}_{10}$, 79.4\%), requiring $2.4\times$ precision lift.

\textbf{Asymmetric risk.} Losses ($\mathcal{S}_{10}$, 304 samples) outnumber gains ($\mathcal{S}_{01}$, 79 samples) by $3.8\times$. The router must be highly selective when choosing the generalist.

\subsection{When Would Extraction Rate Increase?}

From Theorem~\ref{thm:routing} and Proposition~\ref{prop:linear}, the extraction rate $\eta$ increases when:

\begin{enumerate}
    \item \textbf{Larger oracle headroom.} A GUI-trained generalist (e.g., fine-tuning Qwen3.5-27B on GUI data) would increase $n_{01}$, making the minority class easier to detect.
    \item \textbf{Stronger correlation.} Models trained with the zoom pipeline (rather than applied to it post-hoc) may produce better-calibrated consistency signals ($|\rho| > 0.2$).
    \item \textbf{More symmetric disagreement.} If $n_{01}/n_{10}$ approaches 1 (models of similar accuracy with different failure modes), precision $> 1/2$ is easier to achieve.
\end{enumerate}

\section{Limitations}

\begin{enumerate}
    \item The improvement is not statistically significant at $\alpha = 0.05$. The effect size (Cohen's $h = 0.02$) is negligible.
    \item The router doubles inference cost (4 forward passes vs. 2).
    \item The oracle extraction rate (16.5\%) is low, bounded by the generalist's 60.9\% standalone accuracy.
    \item The consistency signal ($|\rho| = 0.13$) is weak. Models trained on the zoom pipeline may exhibit artificially low consistency even when wrong.
    \item The router regresses on 4 of 26 applications (up to $-$4.5\%).
\end{enumerate}

\section{Conclusion}

We formalized \emph{zoom consistency} as a free confidence signal in multi-step GUI grounding pipelines and proved it is proportional to step-1 spatial error. The signal correlates significantly with accuracy ($p < 10^{-6}$) and enables cross-model routing without calibration. On ScreenSpot-Pro, the consistency router captures 16.5\% of the oracle headroom between KV-Ground-8B and Qwen3.5-27B, achieving 80.9\%. The improvement is directionally consistent (92.3\% bootstrap probability) but does not reach conventional significance thresholds with the current model pair. Our theoretical framework (Theorem~\ref{thm:routing}) identifies the bottleneck: the generalist model's low standalone accuracy limits the oracle headroom available for extraction. A stronger generalist would increase both the headroom and the extraction rate, making the zoom consistency signal more valuable.
